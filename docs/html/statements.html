Statements

Control

There are the usual flow-of-control statements:

if name == "anyone" print-line: "Anyone" if x == 0 return 0 else return x * (factorial: x - 1) return # No value given, so this returns "nil". loop token = next-token if token type == 'eof' break consume-token if token type == 'comment' continue while which-item < num-items print-line: (items at: which-item) which-item += 1 for item in items if item name is-empty continue print-line: item name try event-loop catch Exception print-line: exception message throw MessageException new: "Couldn't connect."
Expression Statements

An expression on a line by itself also constitutes a statement:

print-line: "Hello world!"
Assignment

Assignment in Trylon is designed to be easy to read (and type), but the rules behind it can be a little complex.

class MyObject fields total grand-total = 0 # A class variable, with its initial value. reset total = 0 # Sets the object's "total" field. frobnicate: value massaged-value = value * 7 + 3 # Creates a new local variable. total = total + massaged-value # Sets the object's "total" field. grand-total = grand-total + massaged-value # Sets the class variable.

The basic idea is that an assignment will assign to a variable in the current scope, and if no such variable exists, a new local variable will be created.

Assignment can also be a shorthand for a function call:

(array at: 7) = "hello" # is the same as: array at: 7 set-to: "hello"

If the function being set is a unary function, rather than a keyword function, then the setter function's name is the unary function's name with a colon appended:

class MyClass fields real-name name return real-name name: new-name real-name = new-name + " Jr." obj = MyClass new obj name = "Joey Jo Jo Shabbadoo" print-line: obj name # Prints "Joey Jo Jo Shabbadoo Jr."
